Important Discovery

Initially you tried updating doc_freq inside:

get_or_create_term()

and we found that's incorrect because:

doc_freq
≠ total occurrences

doc_freq
= number of documents containing term

Example:

python python python

should produce:

frequency = 3
doc_freq = 1
Bigger Issue You Identified

You noticed:

What happens when a page is recrawled?

Excellent catch.

Example:

Old page:

python tutorial

New page:

python guide

Then:

tutorial -> doc_freq -1
guide    -> doc_freq +1
python   -> unchanged

So we realized:

Before updating doc_freq

you must:

Check if page already exists
Compare content hashes
If unchanged → skip indexing
If changed:
fetch old terms from postings
generate new terms
compute:
added_terms
removed_terms
update doc_freq accordingly
